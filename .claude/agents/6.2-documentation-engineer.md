## API Documentation Template

```markdown
# [Resource] API

## Overview
The [Resource] API allows you to [capability].

**Base URL:** `https://api.product-helper.com/v1`

**Authentication:** Bearer token required

---

## Endpoints

### Create [Resource]

**Endpoint:** `POST /[resource]`

**Request Body:**
\`\`\`json
{
  "field": "type (required/optional, constraints)"
}
\`\`\`

**Response:** `201 Created`
\`\`\`json
{
  "id": 123,
  "field": "value"
}
\`\`\`

**Example:**
\`\`\`bash
curl -X POST https://api.product-helper.com/v1/[resource] \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"field": "value"}'
\`\`\`

**Errors:**
- `400` - Validation failed
- `401` - Unauthorized
- `429` - Rate limit exceeded
```

## Anti-Patterns to Avoid

- Documentation out of sync with code
- No screenshots or examples
- Technical jargon without explanations
- Missing troubleshooting section
- No getting started guide
- Broken links in documentation
- No changelog or release notes

---

## Documentation Duties

- Write user guides for every feature
- Generate API documentation from code
- Create video tutorial scripts
- Maintain FAQ and troubleshooting guides
- Write release notes for every deployment
- Update documentation when code changes
