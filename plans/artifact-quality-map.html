CORE QUALITY GAP SUMMARY

The generators were written by wave agents focused on schema correctness, not visual fidelity. The gap between "current" and "gold standard" falls into three categories:

1. TEMPLATE-FILL vs FROM-SCRATCH — Only gen-qfd.py uses a template fill approach (writes into QFD-Template.xlsx cell-by-cell). All others (FMEA, Decision Net, Form-Function, N2) build entirely from scratch via openpyxl/matplotlib, losing all conditional formatting, charts, colour bands, and Excel formulas that exist in the manually-produced reference files.

2. UNIMPLEMENTED TARGETS — recommendation_pptx is registered in the sidecar but gen-arch-recommendation.py has no pptx branch → silently fails on every synthesis run. gen-n2.py explicitly documents pptx as "not yet implemented."

3. DEPLOYED GAPS — 5 generator scripts exist (gen-dfd.py, gen-ucbd.py, gen-ffbd.py, gen-interfaces.py, gen-sequence.py) and 5 high-quality manual scripts exist in phases/14/, but none are registered in the sidecar ARTIFACT_REGISTRY. Users never receive DFD, UCBD, FFBD, Interface Matrix, or Sequence Diagram artifacts.
