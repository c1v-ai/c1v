## The 5 corrections (locked rulings)

### Correction 1 — P10 absorbs into Wave E as Path B (greenfield refactor)

**The architectural argument** (Rosetta-grounded, locked):

The Rosetta §9 insight: **KB-9 Atlas is an orthogonal substrate — read FROM at M4, M6, M7.b, T6. Not a feeder pipe.** §9.6 explicitly argues against making it a row in §1: *"The Atlas doesn't happen AT a point — it's read FROM at four points. Treating it as a row would either duplicate it across four rows or imply a single anchor phase that doesn't exist."*

P10's two paths:
- **Path A — stub-population nodes (~1-2 weeks).** Treats "no stub" as a sequence problem; adds 7 NEW upstream `populate_X_stub` nodes to feed the 7 existing re-validators. Result: **14 nodes for 7 logical operations.** Same architectural mistake as Cornell's "1-step Decision Matrix" inverted — exploding a single sound concept (the generator) into a 2-step pipe for zero information gain.
- **Path B — greenfield generator refactor (~3-4 weeks standalone, +5 days when absorbed into Wave E).** Each agent reads FROM the substrate (intake, upstream artifacts, FMEA-v1, KB-9 priors) the way M4's decision network reads from M5's morphological matrix. **One node, one job, owns its derivation chain end-to-end.** Aligns with Wave E's deterministic-rule-tree-first narrative.

**Locked: Path B, absorbed into Wave E (NOT a separate v2.1.2 track).**

Why absorption is more efficient than Path A or split-tracking:
1. **Half the node count post-Wave-E.** 7 generator nodes vs 14 (with Path A's stub-populators).
2. **`EvalContext` (G4 ContextResolver) resolves naturally** from substrate reads (intake + upstream artifacts) rather than from artificial "stub artifacts" Path A would produce.
3. **"Why this value?" provenance panel renders ONE chain per decision** (the generator's derivation chain) rather than stitching stub-populator-logic + re-validator-refinement.
4. **One portfolio narrative, not two.** Path A would force telling the engine-first story twice: "first we wrote stub-populators, then we wrote the engine." Path B-absorption is one story: "the engine IS the generator."
5. **`kb-rewrite` + `engine-stories` are already touching these 7 agents** — marginal cost is small (+5 days vs +1-2 weeks standalone Path A).
